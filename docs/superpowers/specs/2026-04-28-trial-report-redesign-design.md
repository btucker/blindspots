# Trial report redesign — design

**Date**: 2026-04-28
**Status**: implemented in branch `improve-html-report`; ready for spec review and commit

## Background

The HTML report generated by `scripts/generate_trial_report.py` had three problems:

1. **Vocabulary collision in the journey chart.** The chart had two overlapping
   axes of meaning: categorical reactions (`confusion`, `frustration`,
   `surprise`, `delight`, `indifference`, `anxiety`) and a 1–5 valence tier
   scale (`alarm`, `tension`, `neutral`, `positive`, `delight`). "Delight"
   appeared in both. "Anxiety" and "frustration" both mapped to score 1
   ("alarm"), collapsing distinct emotions.

2. **Layout couldn't carry detail.** A horizontal 22-step strip with rotated
   tick labels at the bottom either overlapped the chart or required so much
   truncation that nothing useful was visible at a glance.

3. **Reactions read clinically.** The schema (`What happened` /
   `Why it matters`) yielded third-person observation bullets, not a
   first-person playthrough. Hard to feel the persona's experience.

A separate, narrower issue: the comparison table couldn't fit horizontally
inside the report container, and the discovered specs section was using prime
real estate even though it's a reference artifact, not a finding.

## Goals

- Drop the valence/tier vocabulary entirely; reactions stand alone as
  categories.
- Make each step in the journey legible enough to read as a story — heading,
  what the persona thought and did (their voice), and a one-line takeaway.
- Make the comparison the prominent finding section; tuck reference artifacts
  (journal, discovered specs) into collapsed details below.
- Use a heading typeface that's readable at scale (not the wavy italic
  Fraunces display variant).

## Design

### Reaction schema

`reactions.md` becomes the single source of truth. Each entry has:

- **Title** (`## …`) — the heading for the moment (e.g. `Match scores have no scale or explanation`).
- **Reaction** — one of the six categories (drives the color treatment).
- **Narrative** — 3–6 sentences in the persona's first-person voice. What
  they saw, what went through their head, what they tried, how it felt. Use
  contractions, ellipses, asides — sound human. The narrative implicitly
  carries both "what happened" and "how I felt" because it's narrated, not
  observed.
- **Why it matters** — one short analytical takeaway. The narrative carries
  emotion; this line steps back and labels the issue.
- **Screenshot** — filename in `screenshots/` (with `.png`/`.jpg` extension)
  if captured.

For backward compatibility, the parser accepts legacy entries where the
narrative was recorded as `What happened` (the old field name).

The six reaction categories and their colors are unchanged:

| Category     | Color     |
|--------------|-----------|
| delight      | `#15803d` |
| surprise     | `#0e7490` |
| indifference | `#525252` |
| confusion    | `#b45309` |
| anxiety      | `#6d28d9` |
| frustration  | `#b91c1c` |

### Journey section layout

The horizontal arc chart is gone. The journey is rendered as a single-column
vertical stream of action cards. Each card:

- Has a left-edge color border in the reaction's color.
- Begins with a header row: a numbered ring (`01`, `02`, …) outlined in the
  emotion color, plus the emotion category as a small monospace label.
- Heading underneath in sans-serif.
- Narrative as flowing body prose.
- Inline screenshot if attached.
- A "Takeaway" pull quote at the bottom: italic serif text on a tinted
  background (the emotion color at 6% opacity), no left border.

The information density is high enough that a separate emotion chart isn't
needed — color, ring, and label all carry the categorical signal, and the
vertical sequence carries time.

### Section order and section numerals

| §   | Section          | Treatment                                           |
|-----|------------------|-----------------------------------------------------|
| §I  | Journey          | Vertical action-card stream (described above)       |
| §II | Comparison       | Wide bleed-out section so tables fit at full width  |
| §III| Reference        | Two collapsed `<details>`: Exploration journal, Discovered specs |
| §IV | Screenshots      | Bottom strip of all captured screenshots            |

The standalone "Reactions" section is gone (the journey IS the reactions).
The standalone "Discovered specs" and "Journal" sections are folded into
the Reference area, both collapsed by default.

### Comparison bleed-out

The report container stays at `max-width: 980px` so prose reads at a
comfortable measure, but the comparison section uses a `bleed` class that
breaks out:

```css
section.bleed {
  width: min(1240px, calc(100vw - 32px));
  margin-left: 50%;
  transform: translateX(-50%);
}
```

The card inside uses `prose-wide`, which removes the 68ch reading-width cap
and tightens table padding so 3-column comparison tables fit horizontally.

### Typography

Headings switch from Fraunces (italic display serif) to Manrope (sans):

- `h1.display`, `.lede`, `.section-head h2`, `.prose h1/h2/h4` — all sans now.
- `.section-head .numeral` (small `§I`, `§II` markers) — keeps Fraunces for
  editorial flavor at small size.
- `.takeaway p` — keeps Fraunces italic for the pull-quote treatment.

### Agent prompt updates

Both `agents/user-trial-explorer-browser.md` and
`agents/user-trial-explorer-terminal.md` were updated:

- The reactions template no longer asks for `What happened` (clinical
  one-liner). It asks for `Narrative` — 3–6 sentences in first-person voice,
  with concrete coaching: "Use contractions, ellipses, asides — sound
  human. Don't analyze inside the narrative — narrate."
- `Why it matters` is preserved as a one-sentence analytical takeaway.
- Browser agent: still notes the convention for saving screenshots into
  `<output-dir>/screenshots/` with sequential filenames so the report
  generator can embed them.

## Files changed

- `scripts/generate_trial_report.py` — rendering, parsing, CSS.
- `agents/user-trial-explorer-browser.md` — reactions template.
- `agents/user-trial-explorer-terminal.md` — reactions template.

## Trade-offs and notes

- **Loss of the bird's-eye chart.** The previous horizontal chart let the
  reader scan the emotional shape of the trial in one glance ("mostly
  confusion with two delights"). The new vertical stream is denser and more
  readable per step but requires scrolling. Mitigation: the persona's chip
  row at the top of the report (`confusion · 10`, `frustration · 4`, …)
  still gives a quick distribution summary.

- **Existing trial data still uses `What happened`.** Old `reactions.md`
  files render correctly via the legacy fallback in the parser, but they
  read clinically because the agent wrote them in third-person. Re-running
  the trial under the new prompt produces first-person narratives.

- **Bleed-out symmetry.** When the comparison section renders wider than
  surrounding sections, there's a visible width step at section boundaries.
  Acceptable: the comparison is genuinely a different artifact (a wide
  table) and the visual break signals it.

- **Image embedding.** Screenshots are still inlined as base64 data URIs;
  embedded inside action cards as well as in the bottom Plates strip.

## Out of scope

- Connecting line / sparkline summary at the top of the journey (would
  restore the at-a-glance shape view).
- Hover sync between cards and an optional summary chart.
- Mobile fine-tuning beyond the existing breakpoint (`max-width: 720px`).
